# Cloudflare Access для Vlogme Project Hub

Реальная авторизация должна быть настроена перед статическим сайтом через Cloudflare Access. React-приложение не защищает доступ само по себе и не содержит allowlist.

## Что настроить вручную

1. Домен или поддомен Hub должен обслуживаться через Cloudflare.
2. В Cloudflare Zero Trust создайте организацию.
3. Создайте Self-hosted application для адреса Hub.
4. Подключите Identity Provider:
   - Google, если команда входит через Google;
   - или One-time PIN на рабочую почту.
5. Создайте Allow policy по точным рабочим email.
6. Запретите доступ всем, кто не попал в Allow policy.
7. Рекомендуемая длительность сессии: 12-24 часа.
8. Проверьте вход с разрешённой почты.
9. Проверьте отказ для неразрешённой почты.
10. Проверьте выход через `/cdn-cgi/access/logout`.

## Что делает React

В production приложение запрашивает:

```text
/cdn-cgi/access/get-identity
```

Если Cloudflare Access вернул данные, в шапке показывается email пользователя и ссылка выхода. Эти данные используются только для отображения, не для защиты доступа.

В локальной разработке приложение продолжает работать без Cloudflare и показывает метку `Локальный режим`.

## Что запрещено

- Не добавлять форму email/пароль в React.
- Не хранить пароли, токены или OAuth client secret в проекте.
- Не хранить список разрешённых email как механизм защиты frontend.
- Не добавлять реальные email в публичную документацию, если это не согласовано отдельно.
